# Практическая работа №8  
## Разработка гибридного приложения для CPU и GPU

## Общая информация

В рамках практической работы №8 была реализована обработка массива данных с использованием **CPU**, **GPU** и **гибридного подхода CPU + GPU**.  
Работа выполнена на языке **C++** с применением технологий **OpenMP** и **CUDA**.

Все задания выполнялись и тестировались в среде **Google Colab**.

## Среда выполнения

- **Среда:** Google Colab  
- **Язык программирования:** C++  
- **Технологии:**
  - OpenMP — параллельные вычисления на CPU
  - CUDA — параллельные вычисления на GPU
- **Компилятор:** `nvcc`
- **Архитектура GPU:** `sm_75`

---

## Структура проекта

В рамках практической работы реализованы **4 задания**, каждое в отдельном файле:

pr8_task1.cu
pr8_task2.cu
pr8_task3.cu
pr8_task4.cu


---

## Задание 1  
### Обработка массива на CPU с использованием OpenMP  
**Файл:** `pr8_task1.cu`

Выполненные действия:
- Создан массив размером `N = 1 000 000`
- Массив инициализирован значениями `1`
- Реализована параллельная обработка массива на CPU с использованием **OpenMP**
- Каждый элемент массива умножается на `2`
- Выполнен замер времени выполнения
- Для проверки корректности выведены первые 5 элементов массива

---

## Задание 2  
### Обработка массива на GPU с использованием CUDA  
**Файл:** `pr8_task2.cu`

Выполненные действия:
- Массив данных скопирован с CPU на GPU
- Реализовано CUDA-ядро для обработки массива
- Каждый элемент массива умножается на `2`
- Обработанные данные скопированы обратно на CPU
- Выполнен замер времени выполнения на GPU
- Выведены первые 5 элементов массива для проверки результата

---

## Задание 3  
### Гибридная обработка массива (CPU + GPU)  
**Файл:** `pr8_task3.cu`

Выполненные действия:
- Исходный массив разделён на две части:
  - первая половина обрабатывается на CPU (OpenMP)
  - вторая половина обрабатывается на GPU (CUDA)
- CPU и GPU работают параллельно
- Выполнено объединение результатов
- Замерено общее время гибридной обработки
- Выведены первые 5 элементов массива для контроля корректности

---

## Задание 4  
### Анализ производительности  
**Файл:** `pr8_task4.cu`

Выполненные действия:
- Последовательно выполнена обработка массива:
  - на CPU
  - на GPU
  - в гибридном режиме
- Замерено время выполнения для каждого подхода
- Выполнено сравнение производительности CPU, GPU и гибридного режима
- Результаты выведены в консоль

---

## Запуск программ в Google Colab

Все программы компилировались и запускались в Google Colab с использованием `nvcc`.

Пример запуска **задания 4**:

```bash
!nvcc pr8_task4.cu -o pr8_task4 -arch=sm_75 -std=c++11
!./pr8_task4
```

Для остальных заданий используется аналогичная команда, меняется только имя файла.

Пример результатов выполнения
- CPU processing completed. Time: 2.74 ms
- GPU processing completed. Time: 0.08 ms
- Hybrid CPU+GPU processing completed. Time: 1.83 ms
- First 5 elements: 2 2 2 2 2

Краткие выводы:
- Реализованы три подхода обработки массива: CPU, GPU и гибридный
- GPU обеспечивает наименьшее время выполнения за счёт высокой степени параллелизма
- Гибридный подход показывает выигрыш по сравнению с CPU, но уступает чистому GPU
- Корректность работы подтверждена совпадением результатов вычислений
